Sitemap-Max-Files

The Sitemap-Max-Files header can optionally be set on the root node to control the maximum number of sitemap files that be created.

This field is optional. The default value is 500.

The Sitemap-Max-Files header only has an effect when the Sitemap-URL header is set, and the Sitemap-Index header is set to true.

The Sitemap-Max-Entries header controls the maximum number of entries that will be added to an individual sitemap file.

See the RSS, Sitemap, robots.txt page for additional details about how to generate a sitemap.
